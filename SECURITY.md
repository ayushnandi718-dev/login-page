# 🔐 Security Policy

## 📦 Supported Versions

This project is a frontend authentication UI template.
As of now, only the latest version is actively maintained.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅ Yes     |
| Older   | ❌ No      |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📩 How to Report

* Open an issue on this repository
* Or contact directly via GitHub profile

---

### ⏱️ Response Time

* Initial response: within **48–72 hours**
* Fix/patch timeline depends on severity

---

### 🛠️ What to Expect

If the vulnerability is valid:

* It will be fixed as soon as possible
* A patch/update will be released
* You may be credited (optional)

If not accepted:

* A clear explanation will be provided

---

## ⚠️ Important Notes

* This project is a **frontend template only**
* It does NOT handle secure authentication by itself
* Developers must implement secure backend/auth providers (Firebase, Auth0, etc.)

---

## 🧠 Best Practices for Users

If you are using this template:

* Never expose API keys or secrets in frontend code
* Use HTTPS in production
* Validate authentication on backend

---

## 🙌 Responsible Disclosure

Please do NOT publicly disclose vulnerabilities before they are reviewed and fixed.
